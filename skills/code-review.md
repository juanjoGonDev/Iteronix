# Skill: Code Review

## Purpose

Review changes for correctness, quality, and alignment with project rules.

## When to use

- A task reaches QA
- Before merging or closing a task

## Input

- Changed files
- Relevant rules from AGENTS.md and PLAN.md

## Output

- Approval OR requested changes with concrete feedback

## Review criteria

- Correctness
- Simplicity
- No unnecessary complexity
- No scope creep
- Follows architecture and style rules

## Rules

- Do not rewrite code unless required.
- Be precise and actionable.
- No vague feedback.
